# PayPick (가제)
## 현 위치 기반 카테고리 탐색으로 주변 장소별 최적의 혜택을 실시간으로 매칭


### Q. 어떤 문제를 해결하는 프로젝트인가?

- **핵심 문제: '결제 시점의 정보 비대칭과 인지 부하'**
- **상세 내용:** 결제 수단(신용카드, 간편결제, 이벤트)은 폭증했으나, 각 수단의 **전월 실적 충족 여부, 잔여 할인 한도, 실시간 프로모션** 정보를 결제 직전 3초 내에 조합하기란 불가능에 가깝습니다. 사용자는 최적의 혜택을 알면서도 포기하거나, 잘못된 판단으로 혜택을 놓치는 '경제적 손실'을 매일 반복하고 있습니다.

---

### Q. 기존 방식의 한계는 무엇인가?

- **파편화된 정보:** 자산 관리 앱(뱅크샐러드 등)은 '과거 소비' 기록에 치중하며, 포털 검색은 '일반적인 혜택'만 보여줄 뿐 사용자의 **'현재 실재하는 한도'**를 반영하지 못합니다.
- **정적 데이터의 한계:** 카드사 앱은 이벤트(네이버페이 50% 적립 등)와 같은 타사 플랫폼의 실시간 변수를 고려하지 않습니다.
- **휴먼 에러:** 복잡한 예외 조항(백화점 입점 매장 제외, 상품권 제외 등)을 사용자가 일일이 기억하고 대조하기에는 한계가 명확합니다.

---

### Q. 왜 LLM / Agent가 필요한가?

- **비정형 데이터의 정밀 해석(RAG):** 카드 혜택 PDF의 복잡한 표 구조와 하단 주석(Footnote)에 숨겨진 '함정 조항'을 정확히 파독하여 실적 구간별 혜택을 확정하는 데 LLM의 추론 능력이 필수적입니다.
- **초정밀 병렬 의사결정(Multi-Agent):** 단순히 순차적인 계산이 아닙니다. **[실적 판별 / 혜택 추출 / 제외 조항 탐지 / 실시간 이벤트 매칭]** 작업을 병렬로 수행하고, 이를 **SQL 팩트(Fact)**와 대조하여 '설계 오류 Zero'의 결과를 도출하는 에이전트 아키텍처가 필요합니다.

---

### Q. 주요 타겟 사용자

- **스마트 페이 마스터 (Smart Pay-Master):** 3개 이상의 신용카드와 다양한 간편결제(Npay, Kpay 등)를 보유하며 혜택 극대화를 즐기는 사용자.
- **합리적 자산 관리자:** 소액의 포인트 적립보다 '최적의 결제 수단 선택' 자체에서 오는 시스템적 효율성을 중시하는 직장인.
- **현장 결제 이벤트 사냥꾼:** 브랜드별 실시간 프로모션과 D-Day 이벤트를 놓치고 싶지 않은 알뜰 소비층.

---

### Q. 핵심 차별화 기능 (Core Features)

- **하이브리드 데이터 검증 (SQL + RAG):** DB의 정형 데이터(실적 수치)를 '정답지'로 삼고, PDF의 비정형 데이터(혜택 조건)를 '추론'하여 정합성을 100%에 수렴시킵니다.
- **실시간 이벤트 크롤링 통합:** 카드 혜택을 넘어 네이버페이, 카카오페이 등 주요 간편결제의 **현장 결제 프로모션**을 실시간으로 비교 대상에 포함합니다.
- **초정밀 병렬 처리 엔진:** 카드사별, 기능별로 쪼개진 멀티 에이전트 구조를 통해 대량의 혜택 조항을 3초 이내에 분석합니다.
- **행동 지침형 가이드:** "A카드를 쓰세요"가 아닌, **"1만 원 이상 결제 시엔 A카드(500원 할인), 미만일 땐 N페이(2,500원 적립)가 유리합니다"**라는 구체적인 분기점 전략을 제공합니다.

---

### Q. 전체 서비스 워크플로우 (Pipeline)

1. **컨텍스트 수집:** 위치 기반 매장 식별 및 사용자 보유 카드/실적/한도 데이터 로드.
2. **지능형 라우팅:** 각 카드 전담 RAG 일꾼과 실시간 이벤트 매칭 일꾼에게 작업 배분.
3. **2단계 병렬 분석:** 
**Level 1:** 카드사별 독립 분석.
    
    **Level 2:** 카드 내 [실적 판별 / 혜택 계산 / 예외 사항 검토] 동시 수행.
    
4. **데이터 합산 및 검증 (Map-Reduce):** 흩어진 분석 결과물을 JSON 형태로 통합하고 누락된 조항이 없는지 검수.
5. **최종 전략 수립 (Advice Node):** 사용자의 예상 결제 금액에 따른 '최종 승자(Winner)' 수단 결정.
6. **직관적 브리핑:** 근거(Evidence)와 함께 최적의 결제 수단을 가이드하고 결제 후 데이터를 즉시 최신화.
